# 1. Processo, Fluxo e Padrões de Software

## O que é processo de software?
É uma metodologia para as atividades, ações e tarefas necessárias para desenvolver um software.

## Atividades Principais do Processo

De acordo com diferentes autores, as atividades fundamentais variam levemente em nomenclatura, mas mantêm a mesma essência:

| :--- | :--- |
| Comunicação | Especificação |
| Planejamento | Projeto e implementação |
| Modelagem | Validação |
| Construção | Evolução |
| Entrega | - |

## Fluxos de Processo

A organização dessas atividades e suas relações definem o fluxo do processo. Abaixo estão os principais modelos:

### Fluxo Linear
As atividades são executadas em sequência.
![Fluxo Linear](../images/image1.png)

### Fluxo Iterativo
As atividades são repetidas em ciclos antes de avançar.
![Fluxo Iterativo](../images/image2.png)

### Fluxo Evolucionário
O software é desenvolvido em versões cada vez mais completas.
![Fluxo Evolucionário](../images/image3.png)

### Fluxo Paralelo
Diferentes atividades ocorrem simultaneamente.
![Fluxo Paralelo](../images/image4.png)

## Padrões de Processo
Descrevem problemas comuns e sugerem soluções baseadas no contexto. Podem ser aplicados em qualquer nível de abstração. Exemplos de abordagens de avaliação:
* **SCAMPI**: Standard CMMI Assessment Method for Process Improvement.
* **CBA IPI**: CMM-Based Appraisal for internal Process Improvement.
* **SPICE (ISO/IEC 15504)**: Requisitos para avaliação de processos.
* **ISO 9001:2000**: Padrão de qualidade genérico.
